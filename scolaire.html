Projet GSB - site d'Avignon

Conception d'une infrastructure virtualisée avec Proxmox, pfSense, VLAN, Active Directory, DNS, DHCP et Wi-Fi invité.

BTS SIO SISR - réseau 172.24.x.0/24
Detail - Projet GSB

Contexte et objectifs

Le projet porte sur le déploiement du nouveau site GSB d'Avignon après la fusion entre Galaxy et Swiss Bourdin. L'objectif est de livrer une infrastructure complète, segmentée, sécurisée et interopérable pour les utilisateurs, les serveurs internes, l'administration et l'accès Wi-Fi visiteurs.

Proxmox pfSense VLAN Active Directory DNS / DHCP Redondance

Infrastructure technique

Schéma réseau du projet GSB : deux pfSense reliés par les switches S1 et S2, avec les postes utilisateurs, les serveurs AD et Web, ainsi que la borne Wi-Fi visiteurs.

Virtualisation

  • Deux serveurs Proxmox hébergent les machines virtuelles nécessaires au site.
  • Chaque hyperviseur utilise trois interfaces réseau distinctes.
  • WAN pour l'accès extérieur, LAN pour les VLAN internes et DMZ pour les services exposés.

Commutation et cœur réseau

  • Deux switches Cisco Catalyst 2960 assurent la commutation.
  • Les VLAN sont configurés sur les deux commutateurs.
  • Une liaison trunk transporte les flux inter-VLAN entre les équipements.
  • pfSense joue le rôle de routeur principal et de pare-feu.

Segmentation réseau

Le plan d'adressage correct du projet est basé sur le réseau 172.24.x.0/24. Chaque service est isolé dans son propre VLAN afin de séparer les usages et limiter les flux sensibles.

Services Windows Server 2022

Active Directory

Deux serveurs Windows Server 2022 sont configurés comme contrôleurs de domaine du domaine GSBLocal4.local.

  • Réplication des objets de l'annuaire entre les deux serveurs.
  • Authentification centralisée des utilisateurs et des postes.
  • Organisation des droits par groupes métiers.

Groupes et administration

  • AdminsDomaine : administration de l'infrastructure.
  • OfficeUsers : utilisateurs locaux d'Avignon.
  • GSBWorker : employés globaux de l'entreprise.
  • Hello My Dir : intégration d'une solution d'authentification moderne.

Services réseau et disponibilité

  • DNS : résolution des noms internes et accès aux services du domaine, dont le wiki interne.
  • DHCP : distribution des adresses IP pour les VLAN 10, 20, 30 et 150.
  • Les 100 premières adresses de chaque VLAN sont réservées aux équipements statiques.
  • La redondance est mise en place sur AD DS, DNS et selon le mode choisi sur DHCP afin de limiter les interruptions de service.
  • En cas de panne d'un serveur, le second peut prendre le relais pour conserver l'authentification et la résolution de noms.

Sécurité et Wi-Fi visiteurs

Sécurisation

  • pfSense porte les passerelles par défaut des VLAN et filtre les flux entre segments.
  • Le VLAN 99 est réservé à l'administration des équipements réseau.
  • L'administration des commutateurs se fait uniquement en SSH après la configuration initiale.

Wi-Fi visiteurs

  • Borne TP-Link EAP110 alimentée via injecteur PoE.
  • SSID de type GSBXX_Visiteurs selon le numéro du local.
  • Le trafic Wi-Fi est isolé dans le VLAN 150.
  • Les visiteurs n'ont pas d'accès direct au réseau interne.

Compétences techniques

  • Configurer des VLAN, trunks et passerelles inter-VLAN.
  • Administrer un domaine Windows Server avec AD DS, DNS et DHCP.
  • Mettre en place une logique de redondance et de sécurisation.
